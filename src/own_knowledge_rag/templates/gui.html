Post-Review Decision Actions

Use these only after the isolated experiment has been built, evaluated, compared, and manually reviewed. They are merge decisions, not normal ingest stages.

Proper merge order: first build the merged candidate, then evaluate and compare that merged workspace, then accept source files, then promote the merged workspace to baseline. Do not promote the isolated workspace unless you intentionally want a baseline that contains only the staged experiment files.

Decision 1 · Build merged candidate

Creates a realistic candidate index from the current baseline plus staged experiment files. After this finishes, run Evaluation and Compare again on this merged workspace.

Decision 2 · Accept reviewed sources

Copies reviewed staged source files into the baseline source directory. Existing files are backed up before overwrite. Use this only after merged-candidate review passes.

Decision 3 · Promote serving index

Backs up the current baseline workspace, then replaces it with the reviewed merged candidate workspace.

# Experiment loop from the GUI
1. Create or update the experiment record.
2. Build into the experimental `work_dir`.
3. Evaluate ingestion health; add a benchmark only when it is representative.
4. Compare baseline vs isolated experiment.
5. If review passes, build a merged candidate from baseline plus staged files.
6. Evaluate and compare the merged candidate.
7. Accept sources into the library, then promote the merged candidate or keep iterating.
